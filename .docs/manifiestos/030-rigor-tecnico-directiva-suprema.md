2. PRINCIPIOS DE PROSA TÉCNICA ABSOLUTA
Determinismo Lingüístico: Toda descripción debe ser unívoca. Se deben evitar adjetivos subjetivos ("rápido", "fuerte", "potente") en favor de métricas o capacidades técnicas ("latencia sub-10ms", "integridad referencial garantizada", "escalabilidad horizontal").
Jerarquía de Componentes: Los elementos se clasificarán según su rol arquitectónico, no por analogías:
Core Fundamentals: Protocolos base y utilidades compartidas.
Orchestration Layer: Lógica de flujo y gestión de estados.
Interface Layer: Implementaciones visuales y de interacción.
Infrastructure Adapters: Puentes con servicios externos y hardware.
Formalismo en el DocBlock: Los pasaportes técnicos (M-003) deben ser redactados con voz pasiva formal y terminología técnica precisa.
3. ESTÁNDARES DE CODIFICACIÓN (RIGOR LATTICE)
Prohibición de Comentarios Narrativos: Los comentarios dentro del código deben explicar el "por qué" de decisiones complejas mediante referencias a especificaciones técnicas, no describir el "qué".
Documentación de APIs: Se utilizará el estándar OpenAPI/Swagger para servicios y TSDoc para librerías, con descripciones de parámetros basadas en tipos de datos y restricciones matemáticas (Zod).
