Objetivo: Erradicar la cultura de "Escritorio con adaptación" para instaurar la Dictadura del Dispositivo Móvil. En el ecosistema RazWrite Core, el desarrollo de aplicaciones (application-shell) y aparatos modulares (modular-units) inicia obligatoriamente en el Viewport más restringido, escalando potencia solo si el hardware lo permite.
1. LA LEY DE LA ARQUITECTURA DE RESTRICCIONES (CONSTRAINED FIRST)
Todo proceso de forja (udev) debe seguir el flujo de Complejidad Ascendente:
Cero Dependencia de Hover: Queda estrictamente prohibido que una funcionalidad crítica dependa del evento onMouseEnter. El sistema asume que el puntero no existe.
Masa Crítica de Descarga (M-035): El paquete inicial de la aplicación debe ser funcional en redes de baja latencia (3G/HSPA+).
Prioridad de Scroll Vertical: La jerarquía de información se proyecta en una sola columna lógica. El scroll horizontal queda relegado exclusivamente a galerías de imágenes o micro-aparatos específicos.
2. INGENIERÍA DE INTERACCIÓN PULGAR (THUMB-CENTRIC)
La interfaz no se "mira", se "toca". El diseño debe respetar la ergonomía del pulgar:
Zonas de Confort: Los elementos de acción (callToAction) deben residir en los dos tercios inferiores de la pantalla.
Objetivos de Contacto (ISO 25010): El área mínima de interacción es de 44x44 píxeles lógicos.
Feedback Háptico Visual: Toda interacción debe emitir una respuesta visual instantánea (micro-animación) en <50ms para compensar la falta de respuesta física del cristal.
3. GOBERNANZA DE EVENTOS TÁCTILES (UDEV OPTIMIZATION)
El desarrollo de lógica de interacción debe priorizar la fluidez del hilo de superficie:
Passive Listeners: Todos los observadores de touchMove y scroll deben ser declarados como { passive: true } para evitar bloqueos en el renderizado de 60fps.
Debouncing Kinético: Las funciones vinculadas al movimiento deben estar debouncedas por el Metabolic Scheduler para no saturar el bus de eventos en dispositivos de gama media/baja.
4. HIDRATACIÓN ADAPTATIVA POR HARDWARE (M-015-C)
El aparato visual debe mutar su fidelidad basándose en el reporte del BioRefinery:
Modo PEAK: Animaciones fluidas, imágenes de alta densidad (Retina/OLED ready).
Modo ECO: Suspensión de gradientes complejos, desactivación de desenfoques (blur) por CSS y reducción de la tasa de refresco de la telemetría conductual.
5. TAXONOMÍA DE COMPONENTES MÓVILES
Queda prohibido el uso de modales tradicionales que cubran parcialmente la pantalla. Se transmutan en:
Bottom Sheets: Para entradas de datos y selectores (M-004-A).
Toasts Forenses: Para notificaciones de estado asíncronas.
Full-Screen Overlays: Solo para procesos transaccionales de QoS 0 (Pagos/Identidad).
6. SOBERANÍA DE PRUEBAS (MOBILE-ONLY AUDIT)
Ninguna unidad modular se considerará estabilizada si no supera la Prueba de Red Hostil:
Simulación de 250ms de latencia.
Limitación de CPU al 25% (Mobile Mid-tier emulation).
Si la interacción falla o se percibe "Jank" (caída de frames), el aparato es de-comisionado para optimización.
